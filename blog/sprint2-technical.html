CSS Class vs ID

A clash of giants

19 November 2020

To the untrained eye, the CSS attributes of Class and ID may appear to have the exact same function. We set an id or class and we attribute a name:


Imagine this, we’re coding in HTML, we want to set a CSS style to our heading so that it appears blue. We could use either of the above, right? Right!

Hold up

What if we wanted to reuse this styling attribute again, for another title on the same page? This is where things get interesting.

  • The Class attribute is designed so that it can be used many times over on the same page. It can be used multiple times on multiple elements - and elements can also use multiple classes!

  • The ID attribute on the other hand is designed to be UNIQUE. It is used to style only one element. Because of this, it should be used sparingly and only on elements that need to appear the same across the board.

So in this case, it would appear that if we want to make something blue, we should create a makeBlue class, rather than an ID. That way, we can use it again and again, to your heart’s content.

This brings us to Specificity.

Browsers decide which CSS style to apply based on order of specificity. In order from least to most specific we have:

  1. Tags
  2. Classes
  3. ID's

Best practice for using these attributes in CSS is to use the lowest degree of specificity first. This allows us to override the styling of an element using a more specific attribute.

So when styling using CSS, use tags first, then class, then ID. And only use ID when you want one element to be unique!
